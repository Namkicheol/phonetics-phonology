# 한글 용어 사용 규칙 — linguistics (영어학 통합 서브노트)

> **`*_study.html`, `*_concepts.html`, OX 퀴즈 해설(`why` 필드), 카드 본문 등 한글이 등장하는 모든 본문에 적용되는 용어 정책.**
> 핵심: 한글 설명은 유지하되, 그 안의 학술 키워드/개념 용어는 영어 원어로 통일한다.
> 본 레포는 음성학·음운론 + 형태론·통사론 개요 + 의미론 + 화용론 + school grammar 통합이므로 도메인별 용어가 광범위하다.
> 임용 관련 모든 레포(testmaster, englisheducation, phonetics-phonology, transformational-grammar, linguistics)에 공통 적용되는 정책. **원본 정책: 전역 `~/.claude/CLAUDE.md`** + **testmaster `docs/한글용어.md`**.

---

## 원칙

1. **한글 설명 자체는 유지** — 수험생을 위한 한글 풀이/요약/맥락 설명은 그대로 둔다. 한글 블록을 통째로 삭제하지 않는다.
2. **키워드 개념 용어는 영어 원어로** — 학술 용어를 한글 번역 조어로 옮기면 어색하므로 영어 원어로 표기한다.
3. **refs(합격자노트·강사 교재 MD·기출 원문) 표현 우선** — refs에 한글 표현이 있으면 그것을 쓴다. 없으면 영어 원어.
4. **혼용 금지** — 같은 개념을 한 페이지 안에서 한글/영어 혼용 금지. 한쪽으로 통일.

## 사용 OK 한글 표현 (자연스러움 / refs 또는 한국 영어학 학계 표준)

### 음성학·음운론

| 한글 | 영어 |
|------|------|
| 음소 / 분절음 / 초분절음 | phoneme / segment / suprasegmental |
| 자음·모음·이중모음·음절 | consonant · vowel · diphthong · syllable |
| 강세 / 리듬 / 억양 | stress / rhythm / intonation |
| 변별자질 / 자연부류 / 최소대립쌍 | distinctive feature / natural class / minimal pair |
| 동화 / 탈락 / 삽입 | assimilation / deletion / insertion |

### 형태론·통사론·의미론·화용론

| 한글 | 영어 |
|------|------|
| 형태소 / 어기 / 어근 | morpheme / stem / root |
| 굴절 / 일치 / 격 / 의미역 | inflection / agreement / case / thematic role |
| 명사구 / 동사구 / 절 / 구 | NP / VP / clause / phrase |
| 능동·수동 / 진행·완료 | active · passive / progressive · perfect |
| 함의 / 전제 / 함축 | entailment / presupposition / implicature |
| 화용 능력 / 발화행위 / 공손 전략 | pragmatic competence / speech act / politeness strategy |

### 자연스러운 한글 일반 설명

`형태소 경계`, `단어 내부`, `의미 협상`, `문맥 의존`, `표면 형태` 등 — 학술 키워드가 아닌 일반 한글 설명은 그대로 사용.

## 사용 금지 한글 번역 조어 → 영어 원어로 교체

> refs에 등장하지 않거나 한국어로 번역하면 어색해지는 학술 용어. 본문에서 단독 사용 시 영어 원어로 통일.

### 음성학·음운론 도메인

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 장자음 | geminate |
| 단일형태소 | monomorphemic word |
| 의무대조원리 | Obligatory Contour Principle (OCP) |
| 관상 자음 / 관상음 | coronal consonant |
| 전방음 | anterior |
| 활음 | glide |

### 형태론 도메인

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 굴절 접사 | inflectional morpheme (또는 inflectional suffix) |
| 파생 접사 | derivational morpheme (또는 derivational suffix) |
| 굴절 (단독) | inflection |
| 파생 (단독) | derivation |

### 통사론 도메인

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 알파 이동 | Move α |
| 의미역 기준 | Theta Criterion (θ-Criterion) |
| 의문사 이동 | wh-movement |
| 핵 이동 | head movement |
| 공범주 | empty category |
| 격 여과 | Case Filter |

### 의미론·화용론 도메인

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 어휘상 | lexical aspect |
| 달성 동사 | achievement verb |
| 활동 동사 | activity verb |
| 완성 동사 | accomplishment verb |
| 상태 동사 | state verb |
| 반복 해석 | iterative reading |
| 함축 / 회화상의 함축 | implicature / conversational implicature |
| 협력 원리 | Cooperative Principle |
| 격률 | maxim |
| 적합성 이론 | Relevance Theory |

### 담화·정보 구조

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 구정보 | given information |
| 신정보 | new information |
| 말미중량 | end weight |
| 구정보-신정보 계약 | given-new contract |
| 정보 초점 | information focus |

> 위 표는 대표 사례. 일반 원칙: **refs에 한글 표현이 없으면 → 영어 원어 그대로.** 도메인별 매핑은 testmaster `docs/한글용어.md`에 누적된다.

## 적용 패턴 예시

### Before (어색한 번역 조어 단독 사용)

```html
<p>달성 동사가 진행형(progressive)과 결합하면 반복 해석을 강제한다.</p>
```

### After (한글 설명 유지 + 학술 키워드 영어로)

```html
<p>achievement verb가 progressive와 결합하면 iterative reading을 강제한다.</p>
```

---

### Before (전체 한글 블록이 어색한 번역으로 도배)

```html
<p>구정보-신정보 계약은 구정보가 신정보 앞에 위치하는 담화 원리이다.</p>
```

### After (한글 블록은 유지, 학술 키워드만 영어로)

```html
<p>given-new contract는 given information이 new information 앞에 위치하는 담화 원리이다.</p>
```

## 위반 즉시 수정 트리거

- `_study.html` / `_concepts.html` 본문 또는 OX 퀴즈 `why`에 위 표의 ❌ 항목이 단독 한글로 등장 → 즉시 영어 원어로 교체.
- 영어 키워드 + 한글 글로스(`achievement verb(달성 동사)` 등) 형태에서 한글 글로스가 어색하거나 영어와 동일 의미면 → 한글 글로스 제거.
- distinctive feature·통사 원리 이름·의미상 분류명을 한글 번역으로 표기 → 영어 원어로 즉시 교체.
- refs 어디에도 등장하지 않는 새 한글 학술 번역어를 도입 → 즉시 영어 원어로 교체.
